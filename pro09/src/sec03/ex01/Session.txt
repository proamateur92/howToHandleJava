9.4 세션의 특징

정보가 서버의 메모리에 저장된다.
브라우저의 세션 연동은 세션 쿠키를 이용한다.
쿠키보다 보안에 유리하다.
서버에 부하를 줄 수 있다.
브라우저 당 한 개의 세션이 생성된다.
세션은 유효 시간을 가진다.
로그인 상태 유지나 쇼핑몰 장바구니 담기 기능 등에 주로 사용된다.

세션 생성 프로세스

브라우저에서 최초로 서버 접속 시 서블릿은 세션 객체를 생성한다. 세션 객체의 세션 id를 클라이언트에게 전달 클라이언트 브라우저는 세션id를 세션 쿠키에 저장.
다시 서버에 접속 시 할당 받은 세션 쿠키의 세션id를 통해 jsessionId로 세션 객체에 접근.
